* Context
`futon3.hx.api` is the CRUD fa√ßade for hypertext artifacts, anchors, and links.
Each public function builds a canonical hx step, runs structural validation via
`futon3.hx.validate`, and then appends the entry to the log-backed store. This
keeps the runtime state and audit trail in sync.

The API enforces basic invariants (artifact IDs required, link types allowed,
anchors present) and normalizes link metadata (`normalize-link`) so callers can
submit minimal payloads. Listing and retrieval functions read directly from the
in-memory store snapshot.
