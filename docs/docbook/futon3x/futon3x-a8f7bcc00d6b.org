* Context
~GET /api/alpha/mc/tensions~ exports structured tension data. A tension is a
structural mismatch between what the devmaps specify and what the mission
inventory covers. Tensions are derived, not asserted — they come from comparing
the devmap component list against the set of missions that address each
component.

Three tension types:

- ~:uncovered-component~ — a devmap component has no covering mission
- ~:blocked-mission~ — a mission can't run due to unmet dependencies
- ~:structural-invalid~ — a devmap failed validation checks
