* Context

The futon stack's invariant skeleton has nine static-binding cognitive
faculties (single-boundary, evidence-per-turn, coverage-ratchet,
family-canary, subsumption-witness, bounded-disposition, single-locus,
reachable-from-boot, state-snapshot-witness). All nine fire as
ok/violation. None of them tracks a *graduated drive* — a continuous
felt-pressure that rises with accumulated unfinished work and dissipates
when the work is closed.

The 2026-05-02 sweep made the gap concrete: 14 repos, 8 dirty
simultaneously, 40 commits required to clear the backlog. Without an
apparatus that tracks accumulating drain across the substrate, the
operator notices the mess only when it has compounded past the point
of easy resolution.

M-bounded-in-flight-state introduces ~metabolic-balance~ as the 10th
cognitive-faculty shape, with ~metabolic-balance/working-tree~ as the
first operational sibling.

* How to use it

** Reading current state

Run the snapshot script:

#+BEGIN_SRC bash
bb /home/joe/code/futon0/scripts/mana-snapshot.bb
#+END_SRC

Output:
- ~/home/joe/code/storage/futon0/mana-snapshot.json~ — JSON with
  ~:per-repo~ (drain per repo), ~:max-tier~, ~:max-pressure~,
  ~:sessions~ (per-session AIF-head data with phase + balance), ~:pool~.

** HUD widget

Load ~mana-hud.el~ and call ~M-x mana-hud-show~ to echo the
single-line summary in the minibuffer:

: ◔ drain advisory P=1.68 / 14 repos

The widget can also be embedded in ~stack-hud.el~ via ~(mana-hud-render)~.

** Modeline lighter

#+BEGIN_SRC elisp
(require 'mana-modeline)
(mana-modeline-mode 1)
#+END_SRC

The lighter renders only in buffers that carry a session-id (claude-repl /
codex-repl buffers do; other buffers render empty). Polls nonstarter every
30 seconds in the background. Modeline render itself never blocks.

** Boot-time invariant fire

Restart ~./scripts/dev-laptop-env~. Boot output gains:

: [metabolic-balance] working-tree load-time check: OK (max-tier=advisory P=1.68)
: [dev] registered metabolic-balance probe-taps (1 family-id)

** Operational readiness

From a Drawbridge nREPL:

#+BEGIN_SRC clojure
(require '[futon3c.logic.operational-readiness :as op-rd])
(op-rd/snapshot)                                ; full table
(op-rd/ready? :metabolic-balance/working-tree)  ; bool
(op-rd/enabled-but-not-firing-cleanly)          ; violators list
#+END_SRC

** Disposition opt-out

Two paths:

1. Author a per-repo ~.futon-disposition.edn~ (schema in
   ~futon3c.logic.disposition-edn~):

   #+BEGIN_SRC edn
   {:in-progress
    {"src/foo.clj" {:reasoning "long refactor in progress"
                    :since "2026-05-03"
                    :review-by "2026-05-31"}}
    :decided-at "2026-05-03T..."}
   #+END_SRC

2. Reference the path from an active mission's body. The
   ~disposition-derive~ namespace walks
   ~holes/missions/M-*.md~ each fire and treats paths under
   non-terminal-status missions as ~:in-progress~ automatically.
   No file required.
