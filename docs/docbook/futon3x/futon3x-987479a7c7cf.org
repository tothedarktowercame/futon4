* Context
The portfolio review is Mission Control's primary output. It scans mission files
across repos, reads devmap wiring diagrams, computes coverage, and produces a
structured assessment of the entire project state.

~POST /api/alpha/mission-control~ with ~{"action": "review"}~ generates the
review. The response includes:

- *Mission inventory*: 72 missions across 7+ repos, with status
  (complete/in-progress/ready/blocked/unknown)
- *Devmap summaries*: structural prototypes with component/edge/port counts
- *Coverage analysis*: which devmap components are addressed by which missions
- *Gaps*: components with no covering mission
- *Actionable items*: what can start given current dependencies

The review is derived from filesystem state — it scans ~M-*.md~ files for
status headers, parses checkbox gates (~[x]~ vs ~[ ]~), reads ~.devmap~ and
~.edn~ files for wiring diagrams, and matches component names against mission
titles using a 3-tier heuristic (exact match, substring, keyword overlap).
