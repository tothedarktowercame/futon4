* Context
~GET /api/alpha/mc/devmaps~ returns structural summaries of all devmap
prototypes. Devmaps are wiring diagrams from ~futon5/data/missions/*.edn~ and
~futon3/holes/*.devmap~ that describe the intended architecture of each
subsystem: components, edges between them, input/output ports.

* Live output
Captured 2026-03-01 from futon3c on port 7070.

#+BEGIN_SRC sh
curl -s localhost:7070/api/alpha/mc/devmaps | jq '.devmaps[0]'
#+END_SRC

Each devmap includes validation status (~devmap/all-valid~, ~devmap/failed-checks~).
The component list is what tension discovery compares against the mission
inventory — an uncovered component here becomes a tension in the tensions
endpoint.

Arxana's devmap browser (~Lab > Devmaps~) fetches this endpoint and renders all
prototypes with state, component count, edge count, and validation status.
