* Context
The older three-column work established the pattern: an invariant check emits a
durable hyperedge, and Arxana renders that hyperedge as a browsable violation.
The structural-law work in futon3c now uses the same route, but with a generic
=invariant/violation= hyperedge shape carrying =rule=, =family=, =domain=, and
=actionability= in the props.

See docbook://futon3x/futon3x-1e14c482efcf for the original cross-column
invariant checking note. This entry records the first live structural-law
examples that can be inspected through the same Arxana surface.

* Query surface

#+BEGIN_SRC sh
curl -s "http://localhost:7071/api/alpha/hyperedges?type=invariant/violation&limit=100"
#+END_SRC

In Arxana:

1. =M-x arxana-browse=
2. Open =Violations=
3. Inspect the rule, domain, entity endpoints, and action label

The browser now accepts both:
- legacy =invariant/<type>= rows from the three-column ingestion pipeline
- generic =invariant/violation= rows emitted by the structural-law bridge

* Example 1: Mission blocker missing

Source shape:
- family: =existence=
- rule: =missing-blockers=
- domain: =mission=
- endpoints:
  - =inv:existence/missing-blockers=
  - =mission-cycle:C1=
  - =mission-obligation:O-x=

This is the cleanest first example because it is concrete and dispatchable: the
mission cycle points at a blocker that is absent from the obligation ledger.
The Arxana row should therefore read as an actionable structural defect rather
than a vague health warning.

Evidence:
- =futon3c/src/futon3c/logic/arxana_bridge.clj=
- =futon3c/test/futon3c/logic/arxana_bridge_test.clj=

* Example 2: Agency entry/exit asymmetry

Source shape:
- family: =phase-ordering=
- rule: =entry-exit-asymmetry=
- domain: =agency=
- actionability: =needs-review=

This example is useful because it is not auto-fixable. It shows that the same
visual surface can carry review obligations as well as dispatchable repairs.

* Example 3: Codex running-session mismatch

Source shape:
- family: =cross-store-agreement=
- rule: =running-session-mismatches=
- domain: =codex=

This is the best example of a residual candidate family appearing on a live
surface. The invariant is real, but the right repair may still require human
review because two stores disagree about the same session fact.

* Why these examples matter
These examples make the structural-law mission visible in the same way that the
older three-column invariants were made visible:

- a law is declared in code
- a violation is materialized as a hyperedge
- Arxana gives the violation a stable inspection surface
- a docbook entry explains what the viewer is looking at

That is enough for a DOCUMENT phase artifact: not a new theory, but a live
inspection path from invariant runner to hyperedge to browser to narrative.
